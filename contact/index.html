联络方式

特性请求与BUG提交

由于目前暂不开放 Reactive Form 源码仓库的访问权限，如果需要提交特性请求或缺陷报告， 请发送邮件至 yuyunlong@jilinxiangyun.com。

邮件标题格式为 [rcfm 特性请求] + 特性概括标题 或 [rcfm 缺陷报告] + 缺陷概括标题， 请按照如下模板整理邮件内容。

特性请求模板

# 此特性应被添加到 Reactive Form 的原因
请简洁清晰地描述无法解决的场景或问题，或现有特性中不足的地方。

# 此特性的理想工作方式
请简洁清晰地希望该特性该如何运作，以及使用方式。

# Demo 链接
如果有 Demo 链接，请附上。

# 其它说明
其它需要补充的内容。

缺陷报告模板

# 版本
请指明发现缺陷的 Reactive Form 版本

# 缺陷描述
请简洁准确描述该缺陷行为。

# 重现步骤
请准确描述重现步骤，如：
1. 点击……
2. 滚动至……
3. ……

# 正确行为
请简洁准确描述期待产生的正确行为。

# 实际行为
请简洁准确描述实际产生的行为。

# 重现链接
最好附上重现Demo链接。

# 执行环境
请指明执行环境，包括操作系统、浏览器版本/NodeJs版本等。

# 其它说明
其它需要说明的内容。

加入我们

Reactive Form 核心贡献者正在招募中，如有兴趣参与到源码、文档、自动化测试等工作中， 欢迎发送邮件至 lihongda@jilinxiangyun.com。

邮件标题格式为 [rcfm 参与贡献] + 姓名，邮件中请介绍个人技能情况， 也可附上个人资料链接，如 Github、LeetCode、Codepen 等。

更多内容请查看 参与贡献。

商务合作

如果有商务合作需求，请发送邮件至 lihongda@jilinxiangyun.com。

邮件标题格式为 [rcfm 商务合作] + 内容概要标题， 邮件中请阐明合作意向、方式、周期等内容。
